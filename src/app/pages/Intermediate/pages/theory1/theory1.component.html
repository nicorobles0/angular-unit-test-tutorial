Introducción

  1. Estructura archivo de tests Servicios

  2. verify()

    3. Se utiliza para verificar que no haya solicitudes HTTP pendientes o no solicitadas después de realizar las pruebas. Después de realizar tus pruebas y simular las solicitudes HTTP, llamar a verify() es una buena práctica para asegurarte de que todas las solicitudes esperadas se hayan realizado y no haya solicitudes adicionales o pendientes que no se hayan verificado.

  3. expectOne()

    4. Se utiliza para esperar y capturar una solicitud HTTP específica durante las pruebas. La función expectOne() se utiliza para definir qué solicitud HTTP se espera y capturarla para poder realizar aserciones sobre ella.

  4. flush()

    5. Se utiliza para proporcionar una respuesta simulada a la solicitud HTTP capturada. Puedes pasarle como argumento los datos que deseas simular como respuesta.

  5. spyOn()

    6. Cuando utilizas spyOn(), puedes "espiar" métodos o funciones específicas para comprobar si se han llamado, cuántas veces se han llamado, con qué argumentos se han llamado y más. Un spyOn() recibe dor parametros: En el primero recibe el objeto en donde se implementará el espia, y en el segundo parametro recibe el método a espiar.

    Además de espiar en los métodos, la función spyOn(), te permite resolver el metodo con alguna una respuesta controlada, valor, un llamado falso o incluso su ejecución normal.

    1. spyOn( component, 'getCharacterList' ).and.callFake(); Permite hacer un llamado falso, es decir, que el metodo se llama pero no se ejecuta. Este recibe un callback, el cual es el código que se ejecuta en lugar del original
    2. spyOn( component, 'getCharacterList' ).and.callThrough(); Permite la ejecución normal del método.
    3. spyOn( component, 'getCharacterList' ).and.returnValue(); Permite hacer una ejecución falsa del método devolviendo un resultado concreto
