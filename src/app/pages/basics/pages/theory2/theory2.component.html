  2. describe() y xdescribe()

    3. La función describe() es la función principal en donde se deben realizar las configuraciones para las pruebas y en donde se implementan todos los test que quieres aplicar a tu clase o componente. Esta función acepta dos argumentos: una cadena que describe el conjunto de pruebas y una función de devolución de llamada que contiene las pruebas relacionadas.

    1. describe("ComponenteAProbar", () => {
    2. /* Declaración de variables */
    3. /* beforeEach afterEach etc... */
    4. /* Test necesarios */
    5. }

    Si a la función se le agrega una x al inicio xdescribe() esto hará que la prueba como tal y todo lo que contiene sea omitido al momento de correr los tests

  3. Configuración Tesbed

    4. El TesBed hace referencia a la utilidad de angular que nos permite crear un módulo de prueba, para allí poder hacer todas las configuraciones para poder probar nuestras clases.

    Dentro de TesBed tenemos la propiedad configureTestingModule, la cual nos permite añadir las propiedades de nuestro módulo, dentro de este podemos definir una estructura similar a la siguiente:

    1. TestBed.configureTestingModule({
    2. declarations: [ExampleComponent],
    3. imports: [ HttpClientTestingModule ]
    4. providers: [{ provide: PLATFORM_ID, useValue: 'browser' }],
    5. )}

    Tesbed crea un módulo para testear nuestros componentes, por lo tanto lo que añadimos en nuetro Tesbed se asemeja al módulo en donde tenemos declarado nuestro componente:

    Declarations: Los componentes , directivas y canalizaciones que pertenecen,
    Imports: Otros módulos cuyas clases exportadas son necesarias para las plantillas de componentes declaradas,
    Providers: Creadores de servicios que este NgModule contribuye a la colección global de servicios
