Primeros Tests

  1. Ejemplo

    2. En este ejemplo se tiene un contador básico con dos botónes los cuales incrementan el valor del contador en +1 y -1. Adicionalmente existe una validación de cuando el contador tiene un valor mayor o igual a 10 es botón de incrementar se deshabilita.

    Existe un tercer botón el cual incrementa el valor del contador en +1 diferido en 3 segundos, este boton se deshabilita hasta que pasen los tres segundos

  2. detectChanges()

    3. se utiliza en pruebas de Angular para notificar al sistema de detección de cambios (change detection) que se deben aplicar los cambios en los componentes y actualizar la vista. En Angular, el sistema de detección de cambios es el encargado de detectar y aplicar los cambios en los datos de los componentes y propagarlos a través de la vista. Esto asegura que la interfaz de usuario se mantenga sincronizada con el estado de la aplicación.

  3. fakeAsync()

    4. Se utiliza para facilitar las pruebas de código asíncrono. Esta función te permite escribir pruebas para código que involucra operaciones asíncronas, como promesas o temporizadores. Al envolver una prueba con fakeAsync(), puedes utilizar ciertas funciones especiales proporcionadas por Jasmine, como tick() y flush().

  4. tick()

    5. La función tick() se utiliza para avanzar el tiempo en milisegundos específicos y simular que han pasado en el entorno de la prueba. Esto es útil cuando se trabaja con temporizadores o promesas que tienen retrasos en su resolución.

  5. nativeElement

    6. Proporciona acceso directo al elemento HTML real del componente dentro de la prueba. Esto te permite interactuar directamente con el DOM del componente y realizar aserciones en su contenido, estilos, eventos, etc.

  6. (done: DoneFn)

    7. se utiliza para indicar que una prueba es asincrónica y debe esperar a que se llame a la función done() para completarse. Cuando necesitas realizar pruebas unitarias de código asincrónico, como simular llamadas a APIs o operaciones que devuelven promesas, puedes utilizar (done: DoneFn) para asegurarte de que la prueba espere a que se completen las operaciones asincrónicas antes de finalizar.
